# 09주차 - 소프트웨어 개발 프로세스

## 1. 프로세스(Process)의 이해

### 프로세스란?

일을 처리하는 과정이나 순서를 의미하며, 주어진 목표를 달성하기 위해 수행되는 일련의 절차이다.

### 프로세스의 목적

- 기존 노하우 전달
- 시행착오 감소
- 빠른 업무 적응
- 작업 가이드 역할 수행

---

## 2. 소프트웨어 프로세스 모델

### 정의

소프트웨어를 어떻게 개발할 것인지에 대한 전체적인 흐름을 체계화한 모델이다.

### 특징

- SDLC(Software Development Life Cycle) 기반
- 순차적인 개발 단계로 구성
- 개발 전 과정을 체계적으로 관리

### 목적

- 소프트웨어 개발 전 과정 정의
- 예산 및 자원 관리
- 고품질 소프트웨어 생산

---

## 3. 주먹구구식 모델

### 특징

- 즉흥적인 개발 방식
- 공식적인 개발 절차 없음
- 설계나 요구분석 없이 바로 코딩 시작

### 사용 사례

- 개인 프로젝트
- 소규모 프로젝트
- 학기 프로젝트 수준

### 장점

- 빠른 개발 가능
- 간단한 프로젝트에 적합

### 단점

- 문서화 부족
- 유지보수 어려움
- 프로젝트 관리 불가능
- 프로그램 구조 품질 저하

---

## 4. 선형 순차적 모델 (폭포수 모델)

### 특징

- Waterfall Model
- 단계별 순차 진행
- 이전 단계 완료 후 다음 단계 진행

### 개발 단계

1. 요구사항 분석
2. 설계
3. 구현
4. 테스트
5. 유지보수

### 장점

- 관리가 쉬움
- 문서화 체계적
- 요구사항 변화가 적은 프로젝트에 적합

### 단점

- 이전 단계로 돌아가기 어려움
- 변경 사항 반영 어려움
- 중간 결과물 확인 어려움

---

## 5. V 모델

### 개념

폭포수 모델에 테스트 과정을 강화한 모델

### 특징

- 각 개발 단계마다 대응되는 검증 단계 존재
- 품질 보증 강화
- 테스트 중심 개발

### 비교

| 폭포수 모델 | V 모델 |
|------------|---------|
| 산출물 중심 | 검증 중심 |
| 테스트 후반 집중 | 전 과정 테스트 |

---

## 6. 진화적 프로세스 모델

### 등장 배경

폭포수 모델의 단점을 보완하기 위해 등장

### 대표 모델

- 프로토타입 모델

---

### 프로토타입(Prototype)

최종 제품 제작 전에 만드는 시제품 또는 모형

### 목적

- 사용자 요구사항 파악
- 사용자와의 의사소통 도구 활용
- 개발 방향 검증

---

### 프로토타입 개발 절차

#### 1단계

요구사항 정의 및 분석

#### 2단계

프로토타입 설계

- 사용자 인터페이스 중심 설계
- 완전한 설계 아님

#### 3단계

프로토타입 개발

- 시제품 제작
- 사용자 요구사항 확인
- 결과 검증

---

## 7. 통합 프로세스 모델 (UP)

### Unified Process

반복적 개발을 기반으로 하는 소프트웨어 개발 모델

### 주요 활동

- 요구사항 정의 (Requirements)
- 분석 (Analysis)
- 설계 (Design)
- 구현 (Construction)
- 테스트 (Test)
- 유지보수 (Maintenance)

---

### UP의 단계

#### 1. 도입(Inception)

- 프로젝트 목표 설정
- 비즈니스 모델링
- 요구사항 정의

#### 2. 구체화(Elaboration)

- 시스템 구조 설계
- 위험 요소 분석

#### 3. 구축(Construction)

- 구현
- 테스트

#### 4. 전이(Transition)

- 실제 사용자에게 배포
- 운영 환경 적용

---

### 비즈니스 모델링

소프트웨어 개발 전에 업무 환경을 분석하는 과정

#### 주요 역할

- 시스템 목적 정의
- 프로젝트 범위 정의
- 업무 프로세스 분석
- 도메인 모델 구축

---

### BPMN

Business Process Model and Notation

업무 프로세스를 시각적으로 표현하는 국제 표준 표기법

### 목적

- 업무 흐름 분석
- 업무 절차 표준화
- 비즈니스 프로세스 시각화

---

## 8. 애자일 프로세스 모델

### 애자일(Agile)

고객 요구 변화에 빠르게 대응하는 개발 방법론

### 특징

- 반복 개발
- 빠른 피드백
- 지속적인 개선
- 고객 중심 개발

---

### 애자일 선언문 핵심 가치

#### 1

프로세스와 도구보다

➡ 사람과 상호작용 중시

#### 2

문서보다

➡ 실행 가능한 소프트웨어 중시

#### 3

계약 협상보다

➡ 고객과의 협력 중시

#### 4

계획 고수보다

➡ 변화에 대한 대응 중시

---

## 9. 스크럼(Scrum)

### 개념

애자일 개발 방법론 중 하나

### 특징

- 팀 중심 개발
- 짧은 개발 주기 반복
- 프로젝트 관리 중심

### 목적

- 생산성 향상
- 빠른 개발
- 지속적인 개선

---

### 스크럼 개발 절차

1. 제품 백로그 작성
2. 스프린트 계획 회의
3. 스프린트 수행
4. 결과물 개발
5. 스프린트 완료
6. 반복 수행

---

## 핵심 정리

> 소프트웨어 개발 프로세스는 소프트웨어를 체계적으로 개발하기 위한 절차와 방법을 의미한다.
>
> 대표적인 모델로는 주먹구구식 모델, 폭포수 모델, V 모델, 프로토타입 모델, 통합 프로세스 모델(UP), 애자일 모델이 있다.
>
> 최근에는 고객 요구 변화에 빠르게 대응할 수 있는 애자일 및 스크럼 방식이 널리 활용되고 있다.
