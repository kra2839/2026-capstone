# 10주차 - AI Agent와 MCP

## 1. AI Agent 개요

### Agent란?

환경을 인식하고 그에 따라 행동하는 모든 개체를 의미한다.

### AI Agent란?

인공지능 기술을 활용하여 환경을 인식하고, 합리적인 판단을 통해 목표를 달성하기 위한 행동을 수행하는 시스템이다.

---

## 2. 기존 LLM의 한계

### LLM의 장점

- 뛰어난 언어 이해 능력
- 자연스러운 대화 가능
- 다양한 문제 해결 가능

### 한계점

#### 학습 데이터에 의존

- 학습된 정보 범위 내에서만 답변 가능

#### 행동 능력 부족

가능한 것

- 텍스트 생성
- 문서 요약
- 질문 답변

불가능한 것

- 이메일 전송
- 파일 수정
- 코드 실행
- 외부 시스템 조작

### 해결 방법

외부 도구(Tools)와 연결하여 실제 행동 능력을 부여한다.

---

## 3. AI Agent의 특징

### 1) 자율성(Autonomy)

- 사람의 개입 없이 스스로 판단
- 독립적으로 행동 수행

### 2) 목표 지향성(Goal-Oriented)

- 목표 달성을 위한 최적의 방법 탐색
- 결과 중심 행동 수행

### 3) 상황 인식(Context Awareness)

- 주변 환경 변화 감지
- 실시간 정보 분석

### 4) 도구 활용(Tool Usage)

- 외부 시스템 활용 가능
- 다양한 서비스와 연동 가능

---

## 4. AI Agent 작동 방식

### 순환 구조

#### 1단계 : 인지(Perception)

- 외부 환경 정보 수집

#### 2단계 : 사고(Reasoning)

- 정보 분석
- 계획 수립

#### 3단계 : 행동(Action)

- 실제 작업 수행
- 외부 도구 사용

#### 4단계 : 평가 및 학습(Evaluation)

- 결과 분석
- 피드백 반영

---

## 5. AI Agent 활용 사례

### 스마트 가상 비서

- 일정 관리
- 업무 지원

### 자율주행 자동차

- 주행 환경 분석
- 주행 의사 결정

### 금융 자동 매매

- 시장 분석
- 자동 거래 수행

### 제조 및 물류 로봇

- 생산 자동화
- 물류 최적화

---

## 6. MCP(Model Context Protocol)

### MCP란?

AI 모델이 외부 데이터나 도구와 안전하고 표준화된 방식으로 연결될 수 있도록 만든 오픈소스 프로토콜이다.

### 개발사

- Anthropic(Claude 개발사)

### 역할

AI 생태계의 USB-C 포트 역할 수행

---

## 7. MCP의 필요성

### 문제점

AI Agent와 외부 도구마다 연결 방식이 다름

### 해결 방법

표준화된 연결 규격 제공

### 효과

- 개발 효율 향상
- 호환성 증가
- 유지보수 용이

---

## 8. MCP의 특징

### 표준화(Standardization)

- 한 번만 구축하면 여러 AI와 연결 가능

### 보안(Security)

- 권한 제어 가능
- 안전한 데이터 접근 지원

### 확장성(Scalability)

- 로컬 파일
- 데이터베이스(DB)
- 외부 API

등 다양한 시스템 연동 가능

---

## 9. MCP 구조

### MCP Host

AI Agent가 실행되는 환경

#### 예시

- Claude Desktop
- Cursor IDE
- Manus AI

---

### MCP Client

Host 내부에서 서버와 통신 담당

### 역할

- 요청 전달
- 결과 수신

---

### MCP Server

외부 도구를 연결하는 서버

### 역할

- 실제 작업 수행
- 결과 반환

---

## 10. Claude Connector

### Connector란?

Claude가 외부 서비스와 연결되어 실제 작업을 수행할 수 있도록 해주는 기능이다.

### 특징

- 단순 대화형 AI를 넘어 업무 수행 가능
- 외부 앱과 직접 연동
- No-Code 방식 지원

---

## 11. Claude Connector 활용

### 연결 가능한 서비스

- Notion
- Gmail
- Google Calendar
- Google Docs
- GitHub

### 가능한 작업

- 일정 등록
- 이메일 초안 작성
- 문서 생성
- 정보 조회

---

## 12. OAuth

### OAuth(Open Authorization)

비밀번호를 공유하지 않고 외부 서비스 접근 권한을 안전하게 부여하는 인증 방식이다.

### 장점

- 보안성 향상
- 사용자 권한 관리 가능
- 안전한 서비스 연동

---

## 13. Connector와 MCP 비교

| 구분 | Connector | MCP |
|--------|--------|--------|
| 대상 | 일반 사용자 | 개발자 |
| 사용 방법 | 클릭 몇 번으로 연결 | 직접 서버 구축 |
| 난이도 | 쉬움 | 상대적으로 어려움 |
| 특징 | No-Code | 높은 유연성 |
| 활용 범위 | 공식 지원 서비스 | 모든 외부 시스템 |

---

## 14. Connector 활용 필요성

### 기존 LLM 문제

- 답변만 가능
- 실제 업무 수행 불가

### Connector 활용 효과

- 여러 앱의 정보 통합
- 업무 자동화
- 생산성 향상

---

- 반복 작업 자동화

#### 서비스 연동

- Slack
- Gmail
- Google Calendar
- Google Docs
- GitHub

#### Claude Code 연동

- 코드 작성
- 테스트
- 디버깅

---

## 핵심 정리

> AI Agent는 환경을 인식하고 스스로 판단하여 목표를 달성하는 인공지능 시스템이다.
>
> 기존 LLM은 행동 능력이 부족했지만, MCP와 외부 도구 연결을 통해 실제 업무 수행이 가능해졌다.
>
> MCP는 AI와 외부 시스템을 연결하는 표준 프로토콜이며, Claude Connector는 이를 쉽게 활용할 수 있도록 지원하는 기능이다.
>
> 앞으로 AI Agent는 다양한 업무 자동화와 지능형 서비스의 핵심 기술로 활용될 전망이다.
